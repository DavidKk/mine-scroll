# 扫雷游戏规范 SPEC v0.1

> 游戏规则与交互的单一事实来源。与实现冲突时以本文档为准。

---

## 1. 概述

经典 Windows 风格扫雷：在隐藏雷区的网格上，通过数字推理找出所有非雷格并翻开。

| 项   | MVP 值                                      |
| ---- | ------------------------------------------- |
| 棋盘 | 可配置（默认 9×9）                          |
| 雷数 | 可配置（默认 10）                           |
| 平台 | 桌面浏览器（鼠标）；移动端（触摸，见 §3.5） |

---

## 2. 格子状态

每个格子在任意时刻处于以下组合之一：

| 维度   | 可能值                                |
| ------ | ------------------------------------- |
| 内容   | 雷 / 非雷（非雷格有邻雷数 0–8）       |
| 可见性 | covered（未翻开）/ revealed（已翻开） |
| 标记   | none / flagged（插旗）                |

**约束：**

- 已翻开格不可插旗、不可再覆盖
- 插旗格左键无效果，不翻开、不取消插旗
- 游戏结束后：展示所有雷位置；已插错旗可区分（MVP 可选：全部雷格高亮即可）

---

## 3. 交互

### 3.1 左键（Primary）

| 场景             | 行为                                        |
| ---------------- | ------------------------------------------- |
| 首次左键         | 触发布雷（见 §4），再执行开格               |
| covered 且未插旗 | 翻开该格                                    |
| 翻开为 0 邻雷    | 自动 flood fill 展开相连 0 区域及边界数字格 |
| 翻开为雷         | 游戏失败，展示所有雷                        |
| 已插旗 / 已翻开  | 无效果（左键不取消插旗）                    |

### 3.2 右键（Secondary）

| 场景     | 行为              |
| -------- | ----------------- |
| covered  | 切换插旗 ↔ 无标记 |
| revealed | 无效果            |

### 3.3 Chord（双线，经典官方能力）

对已翻开且显示数字 **N**（1–8）的格子，当周围 8 格中 **已插旗数量恰好等于 N** 时：

| 触发方式                        | 行为                               |
| ------------------------------- | ---------------------------------- |
| **双击** 该数字格               | 翻开所有相邻、未插旗、未翻开的格子 |
| **左右键同时按下** 在该数字格上 | 同上                               |

**规则：**

- 仅 `playing` 且已布雷后有效
- 旗数不匹配 → 无效果
- 不要求旗插在对的位置；若周围仍有未翻雷 → 失败
- 数字为 0 的已翻开格不适用（无 Chord）

### 3.4 上下文菜单

- 阻止格子区域默认 `contextmenu`，右键用于插旗（非 Chord 时）

### 3.5 触摸（Mobile）

移动端与桌面 **语义相同**，映射如下（详见 `docs/MOBILE-TOUCH-INPUT-PLAN.md`）：

| 触摸操作                      | 等价桌面操作        |
| ----------------------------- | ------------------- |
| 单击格子                      | 左键开格            |
| 双击已翻开数字格（1–8）       | Chord 双线          |
| 按住格子垂直滑动（上/下等价） | 右键插旗 / 取消插旗 |
| 点击底部 SPACE 按钮           | Space 手动卷屏      |

- 输入按视口宽度分离：`≥768px` 仅鼠标；`<768px` 仅触摸，两套不混用
- 滑动与单击互斥：判定为滑动后不触发开格
- 移动端 SPACE 按钮固定在底部 rail，文案为 `SPACE`（纯文字，无图标）
- 桌面端 SPACE 提示仍在危险区上方（contextual）

### 3.6 暂不实现

- 问号标记

### 3.7 棋盘配置

- 页面顶部提供 **难度预设**（初级 / 中级 / 高级）与 **自定义** 行、列、雷数
- 雷数上限：`行×列 − 9`（保留首击安全区）
- 行 5–30，列 5–40；配置写入 `localStorage`，刷新后保留
- 点击「应用并开始新局」或 Canvas 重开按钮 → 按当前配置新局

## 4. 布雷与首次点击安全

1. 新局开始时 **不布雷**（`minesPlaced: false`），雷的位置尚未确定。
2. **首次有效左键**（开格）时：
   - 在除「点击格 + 8 邻格」以外的格子中 **随机** 放置全部雷（Fisher-Yates 洗牌后取前 N 格）
   - 再计算邻雷数
   - **保证：首击格及 8 邻格绝无雷 → 第一步不可能因踩雷出局**
3. 第二步及之后：雷位已固定，正常扫雷，可踩雷。
4. 若可行格不足（9×9/10 雷不会发生），重试布雷；仍失败则报错。

---

## 5. 胜负条件

| 状态      | 条件                                     |
| --------- | ---------------------------------------- |
| `playing` | 未失败且未胜利                           |
| `lost`    | 任意雷格被翻开                           |
| `won`     | 所有非雷格均已翻开（插旗不影响胜利判定） |

胜利时停止计时；失败时停止计时并展示雷区。

---

## 6. HUD 与控件（MVP）

| 元素          | 行为                                          |
| ------------- | --------------------------------------------- |
| 剩余雷数      | `总雷数 − 当前插旗数`，可显示为负（经典行为） |
| 计时器        | 首次左键后开始；胜/负后停止                   |
| 笑脸/重开按钮 | 随时开始新局，重置棋盘与计时                  |

---

## 7. 视觉（MVP 最低要求）

- 未翻开：凸起灰格
- 已翻开：平面浅底；数字 1–8 使用经典配色（蓝/绿/红/紫/栗/青/黑/灰）
- 雷：翻开或失败时可见图标
- 旗：插旗格显示旗标

---

## 8. 边界 Case

| Case                        | 期望                                   |
| --------------------------- | -------------------------------------- |
| 首点即 flood fill 大片 0 区 | 正确展开                               |
| 快速连点                    | 不崩溃；状态一致                       |
| 胜/负后点击格子             | 不修改局面（或仅重开有效）             |
| 重开                        | 完全重置，计时归零，等待首次点击再布雷 |
